Tomo #26 - Patent Box

Rol: monografia para tratar Patent Box con rigor y no como promesa vaga de ahorro fiscal: solo existe si TradeBase genera o adquiere activos intangibles protegibles y ademas los explota o licencia en las condiciones que exige la normativa.
Fecha de corte: 11/03/2026.
Fuente principal: https://sede.agenciatributaria.gob.es/Sede/impuesto-sobre-sociedades/deducciones-beneficios-fiscales-impuesto-sobre-sociedades/reduccion-rentas-determinados-activos-intangibles-patent-box.html

1. Ficha de trabajo

Campo Detalle
Organismo AEAT / Impuesto sobre Sociedades
Naturaleza Regimen fiscal sobre rentas derivadas de determinados intangibles
Cuello de botella TradeBase No basta con tener software; hacen falta activos intangibles explotados en forma compatible con el regimen
Decision actual NO-GO inmediato; vigilar a largo plazo

2. Que se esta decidiendo realmente

Patent Box no premia desarrollar software para uso propio. Premia determinadas rentas derivadas de intangibles cuando la estructura juridica y de explotacion encaja con la norma.

La pregunta correcta no es:

tenemos tecnologia propia

sino esta otra:

tenemos un intangible elegible que genera rentas en una forma que la norma reconoce.

3. Que tendria que cambiar para que fuera real

  • activos intangibles claramente identificables;
  • proteccion o encaje juridico suficiente;
  • y, sobre todo, una explotacion que genere rentas compatibles: licencia, cesion u otra formula valida.

4. Que podria hacerlo plausible en TradeBase

No Tracciona tal como esta hoy, sino algo como:

  • API de datos o valoracion;
  • licencia de plataforma a terceros;
  • motor propio explotado externamente.

Mientras el producto sea solo software usado internamente para vuestro propio negocio, la linea no tiene materialidad real.

5. Que no conviene hacer

  • meter Patent Box en el presupuesto 2026-2027;
  • usarlo como argumento para justificar decisiones tempranas de producto;
  • dar por hecho que cualquier software propio abre el regimen.

6. Que documentacion futura exigiria

  • identificacion del intangible;
  • soporte juridico de titularidad;
  • modelo de explotacion o licencia;
  • trazabilidad de rentas derivadas;
  • coordinacion fiscal seria en IS.

7. GO / NO-GO / NOT YET

GO

Solo si en el futuro existe un intangible explotado externamente en forma elegible.

NO-GO

Mientras el software siga siendo herramienta interna del propio negocio.

NOT YET

Si quereis conservar la opcion conceptual, pero todavia no existe el modelo de explotacion adecuado.

8. Conclusion operativa

#26 es una linea de madurez futura. No debe gobernar decisiones actuales, pero conviene no cerrarla conceptualmente si la arquitectura acaba generando activos licenciables.
